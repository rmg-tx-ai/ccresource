Claude.skills

Reusable knowledge modules stored in .claude/skills/ that extend Claude's capabilities with project-specific, team-specific, or domain-specific information.

üí° Key Benefit
Skills are loaded automatically when relevant, reducing context window overhead and ensuring consistent application of best practices.
  • Stored in .claude/skills/ directory
  • Automatically triggered by context
  • Contains workflows & reference data
  • Manually invokable with /skill-name

Building Effective Claude.skills

üîÑ

Evaluation-Driven Development

  1. Complete task without skill - Work through problem with Claude, noting what context you repeatedly provide
  2. Identify reusable pattern - Recognize workflows, schemas, rules, naming conventions
  3. Ask Claude to create skill - "Create a Skill that captures this pattern we just used"
  4. Review for conciseness - Remove unnecessary explanations Claude already knows
  5. Test with fresh instance - Use skill in new Claude session on similar tasks
  6. Iterate based on observation - Return with specific observations from Claude's behavior
üí° Pro Tip
"Create evaluations BEFORE writing extensive documentation. This ensures your Skill solves real problems rather than documenting imagined ones." - Anthropic Docs

Content Best Practices

  • Be concise - Remove what Claude already knows
  • Use decision trees - Guide through workflows with branching
  • Show examples - Demonstrate desired style more than describing
  • Provide defaults - Give recommended approach with alternatives
  • Make rules prominent - Use "MUST" or "IMPORTANT:" for critical items
  • Use forward slashes - Unix paths work everywhere

Display Modes

In-Process Mode

  • All teammates in main terminal
  • Shift+Up/Down to select teammate
  • Enter to view session, Escape to interrupt
  • Ctrl+T to toggle task list
  • Works in any terminal

Split-Pane Mode

  • Each teammate in own pane
  • View all outputs simultaneously
  • Click into pane to interact
  • Requires tmux or iTerm2

Coordination Patterns

Delegate Mode

Press Shift+Tab to enable delegate mode

Restricts lead to coordination-only tools:

  • Spawning teammates
  • Messaging teammates
  • Shutting down teammates
  • Managing task list
‚ö†Ô∏è Common Issue
Without delegate mode, lead sometimes starts implementing instead of coordinating.

Direct Communication

Message any teammate directly:

  • Give additional instructions
  • Ask follow-up questions
  • Redirect approach mid-task
  • Provide context others lack

In-process: Shift+Up/Down to select, then type

Split-pane: Click into teammate's pane

When to Use Agent Teams

‚úÖ Ideal Scenarios

  • Parallel exploration - Multiple independent research threads
  • Multi-perspective analysis - Same problem, different viewpoints
  • Distributed implementation - Independent features/modules
  • Concurrent testing - Multiple test suites

‚ùå Poor Scenarios

  • Sequential dependencies (B waits for A)
  • Shared state requiring sync
  • Simple linear workflows
  • Tasks needing frequent human input
üí° Team Size
2-4 teammates typically optimal. More increases coordination overhead.

Avoid Over-Engineering ‚ñº
‚ö†Ô∏è Key Principle
"Only make changes that are directly requested or clearly necessary."
  • Scope: Don't add unrequested features
  • Documentation: Don't add docstrings to unchanged code
  • Defensive coding: Don't add error handling for impossible scenarios
  • Abstractions: Don't create helpers for one-time operations

Context Management

Provide Rich Content

  • Reference files with @filename
  • Paste images directly (drag & drop)
  • Give URLs for docs/APIs
  • Pipe in data: cat error.log | claude
  • Let Claude fetch what it needs

Avoid Context Pollution

‚ö†Ô∏è Kitchen Sink Session
Starting with one task, asking unrelated things, returning to first. Context fills with irrelevant info.

Fix: Use /clear between unrelated tasks

‚ö†Ô∏è Correcting Over and Over
Claude wrong, you correct, still wrong, correct again. Context pollutes with failed approaches.

Fix: After 2 failed corrections, /clear and reformulate

Quick Fixes

Context Issues

  • /clear between unrelated tasks
  • After 2 failed corrections, start fresh
  • Use @filename for file references
  • Pipe data directly instead of pasting

Skill Testing

  • Test with fresh Claude instance
  • Use actual tasks, not contrived scenarios
  • Observe behavior, iterate based on usage
  • Return to skill-creator Claude with specifics

Diagnostic Checklist

Skill Not Working?

  1. Check skill is in .claude/skills/
  2. Verify SKILL.md has frontmatter
  3. Test description specificity
  4. Look for over-verbose content
  5. Confirm examples are present

Agent Team Not Coordinating?

  1. Verify EXPERIMENTAL flag is set
  2. Check if lead is in delegate mode
  3. Review task dependencies
  4. Ensure roles are clearly defined
  5. Look for sequential bottlenecks
